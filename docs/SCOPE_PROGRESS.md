### 判断: SEOクリック狙いは打ち切る
- **ボトルネックは「表示されない」ではなく「順位が低くてクリックされない」**。原因はドメインの権威性の低さ（新規＋AI量産→AdSense閉鎖の履歴）。記事を増やしても超えられない壁。
- 高表示クエリ（canva代替145、gpt5.5 89）は**体験がなく正直に書けない**。書けば「体験なし量産」の再演でAdSense閉鎖時と同じ過ち。
- GPT-5記事(7/09生成)を精読 → **中身ゼロのAI量産フィラー**であることを確認。自動投稿は「順位が付かない記事」を量産している構造。
- 体験のあるクエリ（chatgpt claude比較・aiライティング副業等）は表示11〜15回と極小。SEOの天井が元々低い。
- **結論**: 2.5ヶ月・実質1クリックの事実と合わせ、SEO流入を追うのは投資対効果に見合わない。ブログの価値は「検索流入」ではなく「スキルの証明＝ポートフォリオ／集客装置」に置く（メモリ方針を再確認）。

### 決めたこと
- **canva/gpt5のような体験なし記事は作らない**（方針違反・逆効果）
- **自動投稿は止めずそのまま**（Groq無料枠でコスト0、更新実績とポートフォリオの厚みにはなる）。ただし**KPIを「クリック数」にしない**
- SEO微最適化に工数を割かない（該当クエリの需要が体験と噛み合わないため打ち手なし）

## 2026-07-05 自動投稿v5改修 → v5.1で完全復旧

### Search Console通知（7/3受信）

- **原因**: 以前削除した旧記事のURLに設定した301リダイレクト（意図的）をGoogleが検出
- **対応**: **修正不要**。時間経過とともにGoogleのインデックスから旧URLが消えていく正常な過程

- 6/10・6/21に不承認メール2通あり（既知。6/28にAdSense完全撤退済み・アカウント閉鎖済み）

### 自動投稿の失敗原因を特定・修正
- **原因**: Groq無料枠のTPM制限（12,000トークン/分）でレート制限429エラーが頻発
  - 旧v4: プロンプト約9,000トークン + max_tokens 8,000 = 12,000 TPM超過
  - 7/3〜7/4の毎日投稿は全て品質基準未達でスキップされていた
- **v5（2分割）**: Part1+Part2で合計1,950〜2,160文字 → 品質基準2,500文字に届かず
- **v5.1（3分割）で解決**:
  - 記事生成を**3回に分割**（Part1: 導入+H2×1 / Part2: H2×2 / Part3: まとめ+H2×1）
  - 各APIコール間に65秒待機でTPMリセット確保
  - 品質基準: 2,500→**2,000文字**に緩和
  - 結果: **3,272文字**で品質チェック一発OK
- **ワークフロー修正**: `git diff --name-only`→`git status --porcelain`に変更
  - 新規ファイル（untracked）が検出できず、記事生成成功後もデプロイがスキップされていたバグを修正
